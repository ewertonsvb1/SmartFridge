SMARTFRIDGE - PASSO A PASSO PARA GERAR APK E TESTAR NO CELULAR

Este guia considera:
- PC com Windows
- backend rodando na sua maquina
- celular Android na mesma rede Wi-Fi do computador


1. O QUE VOCE PRECISA TER INSTALADO

No computador:
- Java 17
- Maven
- Flutter
- Android SDK

Para conferir:

No PowerShell, rode:

java -version
mvn -version
flutter doctor

Importante:
- hoje a sua maquina esta com Java 8
- este projeto Android usa Java 17 no Gradle
- com Java 8 o APK nao vai gerar


2. AJUSTAR O JAVA 17

Instale o JDK 17 e aponte o ambiente para ele.

Depois confirme:

java -version

O resultado precisa mostrar Java 17.

Se voce tiver mais de uma versao instalada, confira tambem:

echo $env:JAVA_HOME

Se necessario, ajuste o JAVA_HOME para a pasta do JDK 17.


3. SUBIR O BACKEND

O app Android precisa acessar a API do backend.

No PowerShell:

cd F:\Projetos\Geladeira\backend
mvn spring-boot:run

Se tudo subir certo, a API deve ficar disponivel em:

http://localhost:8080

Observacao:
- para teste local simples, pode usar o profile dev com H2
- nao precisa PostgreSQL so para um teste rapido no celular


4. DESCOBRIR O IP DA SUA MAQUINA

O celular nao consegue acessar "localhost" do seu PC.
Ele precisa acessar o IP local da sua maquina na rede.

No PowerShell, rode:

ipconfig

Procure a placa de rede que voce esta usando.
Normalmente sera uma destas:
- Adaptador de Rede sem Fio Wi-Fi
- Adaptador Ethernet

Dentro dela, procure:

Endereco IPv4 . . . . . . . . . . . . : 192.168.x.x

Exemplo real de formato:

192.168.0.15

Esse sera o IP para usar no APK.

Se aparecer mais de um IP:
- use o da rede em que o celular tambem esta conectado
- normalmente comeca com 192.168.x.x ou 10.x.x.x


5. TESTAR SE O BACKEND ESTA ACESSIVEL NA REDE

Com o backend rodando, teste no navegador do seu computador:

http://SEU_IP:8080

Exemplo:

http://192.168.0.15:8080

Depois teste no navegador do celular:

http://SEU_IP:8080

Se abrir no celular, a rede esta pronta.

Se nao funcionar:
- confira se o backend ainda esta rodando
- confira se o IP esta correto
- confira se o firewall do Windows nao esta bloqueando a porta 8080


6. LIBERAR O FIREWALL DO WINDOWS, SE NECESSARIO

Se o celular nao conseguir acessar a API, pode ser bloqueio do firewall.

No Windows:

1. Abra "Firewall do Windows Defender"
2. Clique em "Permitir um aplicativo ou recurso pelo Firewall do Windows Defender"
3. Ou crie uma regra de entrada liberando a porta 8080

Se quiser fazer pela interface avancada:

1. Abra "Firewall do Windows Defender com Seguranca Avancada"
2. Regras de Entrada
3. Nova Regra
4. Porta
5. TCP
6. Porta especifica: 8080
7. Permitir a conexao
8. Marque rede Privada
9. Salve a regra


7. GERAR O APK

Agora gere o APK apontando para o IP da sua maquina.

No PowerShell:

cd F:\Projetos\Geladeira\mobile
flutter pub get
flutter build apk --release --dart-define=APP_ENV=prod --dart-define=API_BASE_URL=http://SEU_IP:8080

Exemplo:

flutter build apk --release --dart-define=APP_ENV=prod --dart-define=API_BASE_URL=http://192.168.0.15:8080

Importante:
- use http://
- nao use localhost
- nao use 127.0.0.1
- o IP precisa ser o do computador, nao o do celular

Observacao importante:
- o projeto ja esta configurado para gerar `release` com a chave de debug quando nao existir keystore propria
- para teste no seu proprio celular, isso e suficiente
- so precisa criar keystore propria quando quiser distribuir versoes de release de verdade


8. ONDE O APK SERA GERADO

Depois do build, o arquivo ficara em:

F:\Projetos\Geladeira\mobile\build\app\outputs\flutter-apk\app-release.apk

Opcao com script pronto do projeto:

cd F:\Projetos\Geladeira
.\scripts\build_mobile_apk.ps1 -ApiBaseUrl http://SEU_IP:8080


9. INSTALAR O APK NO CELULAR

Voce pode instalar de algumas formas.

Opcao A - cabo USB:
- conecte o celular no computador
- copie o arquivo app-release.apk para o celular
- abra o arquivo no celular
- permita instalar apps dessa origem, se o Android pedir

Opcao B - WhatsApp, Telegram, Drive ou e-mail:
- envie o APK para voce mesmo
- abra no celular
- instale


10. ANTES DE TESTAR NO CELULAR

Confirme tudo isso:

- o backend continua rodando no computador
- computador e celular estao na mesma rede Wi-Fi
- voce usou o IP correto no comando do build
- a porta 8080 nao esta bloqueada
- o Java ativo e o 17


11. COMO TESTAR

Abra o app no celular.

Se o backend estiver em dev, voce pode usar a conta padrao:

Login: demo@smartfridge.local
Senha: 123456

Depois teste:
- login
- cadastro
- criacao de produto
- dashboard
- filtros
- lista de compras
- notificacoes


12. SE DER ERRO DE CONEXAO

Os erros mais comuns sao:

1. IP errado
- gere outro APK usando o IP correto do computador

2. Backend parado
- rode de novo:
  cd F:\Projetos\Geladeira\backend
  mvn spring-boot:run

3. Firewall bloqueando
- libere a porta 8080 no Windows

4. Celular em outra rede
- coloque celular e PC na mesma rede Wi-Fi

5. APK gerado com URL errada
- gere novamente com:
  flutter build apk --release --dart-define=APP_ENV=prod --dart-define=API_BASE_URL=http://SEU_IP:8080

6. Java errado
- se `java -version` mostrar Java 8 ou outra versao diferente de 17, o build Android pode falhar


13. RESUMO CURTO

Sequencia minima:

1. garantir Java 17:
   java -version

2. subir backend:
   cd F:\Projetos\Geladeira\backend
   mvn spring-boot:run

3. descobrir IP:
   ipconfig

4. gerar APK:
   cd F:\Projetos\Geladeira\mobile
   flutter pub get
   flutter build apk --release --dart-define=APP_ENV=prod --dart-define=API_BASE_URL=http://SEU_IP:8080

5. instalar:
   F:\Projetos\Geladeira\mobile\build\app\outputs\flutter-apk\app-release.apk

6. testar no celular com a mesma rede Wi-Fi

Opcional:
- usar o script pronto:
  cd F:\Projetos\Geladeira
  .\scripts\build_mobile_apk.ps1 -ApiBaseUrl http://SEU_IP:8080


14. COMANDO MODELO FINAL

Subir backend:

cd F:\Projetos\Geladeira\backend
mvn spring-boot:run

Gerar APK:

cd F:\Projetos\Geladeira\mobile
flutter pub get
flutter build apk --release --dart-define=APP_ENV=prod --dart-define=API_BASE_URL=http://192.168.0.15:8080

Troque 192.168.0.15 pelo IP real da sua maquina.
