# AI Support Map

Esta pasta resume o projeto SmartFridge a partir do codigo atual do repositorio.

Use primeiro:

1. `.ai/core/architecture.md`
2. `.ai/core/conventions.md`
3. `.ai/core/patterns.md`
4. `.ai/context/backend.md`
5. `.ai/context/frontend.md`

Depois leia apenas os contextos modulares relevantes para a tarefa:

- `.ai/context/auth.md`
- `.ai/context/products.md`
- `.ai/context/shopping.md`
- `.ai/context/notifications.md`
- `.ai/context/users.md`

Objetivo desta estrutura:

- reduzir tempo de onboarding de agentes
- evitar solucoes genericas que ignorem o estilo do projeto
- reforcar os fluxos reais entre Spring Boot, JWT e Flutter
- guiar implementacoes consistentes com os endpoints e providers atuais
